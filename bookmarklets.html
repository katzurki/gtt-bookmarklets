Adding Shortcuts to GTT (Merge, Apply to all, etc.)

 

TL;DR: Drag these bookmarklets to the bookmarks bar, then use a shortcut manager to “click” and execute them (for example, the very excellent Shortkeys for Chrome, by Mike Crittenden; don’t forget to tick off Active while in form fields in Activation Settings for each shortcut):

Merge          Apply to all      Copy current hotel name (relevant only to Booking.com translators)

Works in Windows and Google Chrome, and [unconfirmed] OS X and Chrome.

However, two of the most often used actions, Merge Down and Apply to All, can only be accessed by clicking the UI element. For Merge, this requires 4 clicks: Edit -> Merge Segments -> With next segment -> OK, which is a lot of clicking for something that should be at the tips of your fingers—with a keyboard shortcut.

It’s possible to simulate this functionality with a Javascript bookmarklet, which can be mapped to a keyboard shortcut using an appropriate browser extension. Just drag these links to the bookmarks bar, and there you have the bookmarklets.

Merge

Merges the current segment with the next. Saves 4 clicks. My mapping is Alt+S.

 

Apply to all

Propagates current segment translation across identical segments in the document. Saves 2 clicks. My mapping is Alt+A.

Copy current hotel name

Copies the current property source name, deriving it from the editbox position. Note that you still need to paste it by pressing Ctrl+V. My mapping is Ctrl+Shift+V. This copies the hotel name, and you press Ctrl+V to paste it where needed.

In order to map a keyboard shortcut to a bookmarklet, I use Shortkeys for Chrome. The other shortcut managers did not handle clicking correctly, and bookmarklets were failing to fire. Kudos to Mike Crittenden for developing it! Setting it up is more or less self-explanatory: choose a combination, choose Run bookmarklet, specify which bookmarklet, and don’t forget to check “Active in text fields” for each shortcut.

 

Reload the GTT and you’re good to go!
